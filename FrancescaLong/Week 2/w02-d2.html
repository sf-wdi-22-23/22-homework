  • 3 input types and real world examples:

    1. Drop down list - these are often used for lists of states when filling out address forms.

    2. File picker - used to choose pictures to upload to facebook or any other site.

    3. Text area - used for things like name inputs. It can be limited in length to prevent users from going crazy but allows them to enter text relatively freely.

  • Definition of client, server, and API.

    1. Client - A client is the end user computer that an individual uses to access the internet. It is this machine that sends requests for information to the server and receives the data back. The person using the client can read items, fill out forms, follow links, etc. on the websites.

    2. Server - The server hosts the website and any underlying data needed to power the website. It receives requests from the client and provides the new web pages when a link is clicked, provides data when requested by the user / client computer.

    3. API - An API acts as an interface between the client and the server. It runs on the server and makes the data from the client computer digestible by the server or vice versa allowing the two to interact smoothly and quickly.
